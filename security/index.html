Security Architecture

Five Pillars of Government-Grade Security

A layered security posture designed for the real-world threats and compliance requirements California county agencies face.

PII Data Protection

End-to-end protection for personally identifiable information at every stage of the data lifecycle.

  • AES-256 encryption at rest for all stored data
  • TLS 1.3 encryption in transit for all communications
  • Role-based access controls limiting data exposure to authorized users only
  • Full audit logging of all data access and modification events
  • Data minimization principles applied across all product features

Compliance Posture

Designed to support the compliance obligations California county agencies carry, including health and criminal justice data requirements.

  • HIPAA-aware design for health and benefits program data
  • California Consumer Privacy Act (CCPA) alignment
  • CJIS-ready architecture for law enforcement adjacent workflows
  • California state data privacy and retention requirements
  • IRS Publication 1075 alignment for federal tax information

Infrastructure

Hosted on government-specific cloud infrastructure, engineered for availability, reliability, and compliance.

  • Hosted on Microsoft Azure Government Cloud (US Gov Virginia / US Gov Arizona)
  • SOC 2 Type II aligned practices and controls
  • Automated vulnerability scanning and patch management
  • Disaster recovery with defined RPO and RTO objectives
  • 99.9% uptime SLA with monitored availability dashboards

Data Isolation

Strict multi-tenant architecture ensures each county's data remains completely separate and inaccessible to other agencies.

  • Logical data isolation between all county tenants
  • Separate encryption keys per county customer
  • No cross-tenant data access under any operational scenario
  • Documented and tested data boundary controls
  • County-specific data retention and deletion policies

Access Controls

Comprehensive identity and access management aligned with least-privilege principles and government identity standards.

  • Multi-factor authentication (MFA) required for all users
  • Least-privilege access model enforced at the role level
  • Integration with county Active Directory / SSO systems
  • Automated session timeout and idle session termination
  • Privileged access management for administrative functions

Incident Response

Documented, tested incident response procedures designed around the notification and reporting requirements county agencies face.

  • Documented incident response plan aligned to NIST framework
  • 24/7 security monitoring with automated alerting
  • Defined breach notification procedures meeting state requirements
  • Annual penetration testing by independent third parties
  • Security review available upon request for agency IT teams
